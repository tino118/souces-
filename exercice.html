Exercice - Démarrer un projet

Effectué
  • 1 minute

Maintenant que vous avez appris à vous servir des principales commandes Git, vous pouvez passer à la création d’un projet dans Git.

Dans les exercices qui suivent, vous allez commencer à utiliser Git en ajoutant un fichier HTML simple dans votre arbre de travail. Ensuite, vous allez apporter des changements au répertoire et apprendre à les commiter.

Créer et ajouter (indexer) un fichier

Git ne peut pas faire grand-chose avec des répertoires vides. Nous allons donc ajouter un fichier à l’arbre de travail pour qu’il serve de page d’accueil pour le site web de photos de chat.

  1. Vérifiez que votre session Cloud Shell est toujours active et que vous vous trouvez dans le dossier nommé Cats.

  2. Utilisez une commande touch pour créer un fichier nommé index.html :

    touch index.html

    touch met à jour l’heure de la dernière modification du fichier, si le fichier existe. Si le fichier n’existe pas, Git crée un fichier vide avec ce nom de fichier.

  3. À présent, utilisez git status pour connaître l’état de l’arbre de travail :

    git status

    Git répond en vous informant que rien n’a été commité, mais que le répertoire contient un nouveau fichier :

    No commits yet

Untracked files:
  (use "git add <file>..." to include in what will be committed)

    index.html

nothing added to commit but untracked files present (use "git add" to track)

    Notez que git status vous donne des conseils sur ce que vous pouvez faire ensuite. Git peut être configuré pour fournir moins de détails, mais à ce stade, il est préférable d’en avoir.

  4. Utilisez git add pour ajouter le nouveau fichier dans l’index de Git, suivi de git status pour vérifier l’état. N’oubliez pas le point à la fin de la commande. Il indique à Git d’indexer tous les fichiers du répertoire actif qui ont été ajoutés ou modifiés.

    git add .

    Un commit vient donc d’être indexé. L’index de Git est une zone d’indexation pour les commits. Il s’agit de la liste de toutes les versions de fichiers qui feront partie du prochain commit que vous effectuerez.

    Au lieu d’utiliser git add ., vous auriez pu utiliser git add index.html parce que index.html était le seul nouveau fichier présent dans le répertoire. Mais si plusieurs fichiers avaient été ajoutés, git add . les aurait tous traités.

  5. Enfin, utilisez git status à nouveau pour vérifier que vos changements ont été indexés correctement :

    git status

  6. Vous devriez voir un résultat similaire à cet exemple :

    On branch main

Initial commit

Changes to be committed:
  (use "git rm --cached <file>..." to unstage)

    new file:   index.html

Effectuer votre premier commit

Maintenant que index.html a été ajouté à l’index, l’étape suivante consiste à le commiter.

  1. Utilisez la commande suivante pour créer un commit :

    git commit index.html -m "Create an empty index.html file"

    L’indicateur-m de cette commande indique à Git que vous fournissez un message de commit.

    Il existe de nombreuses façons de rédiger un message de commit. Toutefois, il est préférable que la première ligne indique l’effet du commit sur l’arbre. Il est également courant d’utiliser une majuscule pour la première lettre et d’omettre le point de fermeture pour économiser de l’espace. Imaginez que la première ligne du message se termine par « Une fois envoyée, ce commit va... »

    Un message de commit peut comprendre plusieurs lignes. La première ligne ne doit pas comporter plus de 50 caractères et doit être suivie d’une ligne vide. Les lignes suivantes ne doivent pas comporter plus de 72 caractères. Ce ne sont pas des exigences fermes et définitives (elles datent de l’époque des cartes perforées et des terminaux passifs), mais elles rendent la sortie git log plus lisible.

  2. Git répond avec une confirmation de votre action :

    [main (root-commit) 87e874c] Create an empty index.html file
 1 file changed, 0 insertions(+), 0 deletions(-)
 create mode 100644 index.html

  3. Continuez avec la commande git status pour vérifier que l’arbre de travail est « propre », c’est-à-dire qu’il ne contient aucun changement non commité.

  4. À présent, utilisez une commande git log pour afficher des informations sur le commit :

    git log

  5. La réponse de Git doit ressembler à cet exemple :

    commit 87e874c4aeeb3f9692ae5d9875235353708d7dd5
Author: User Name <user-name@contoso.com>
Date:   Fri Nov 15 20:47:05 2019 +0000

Create an empty index.html file

Modifier index.html et commiter le changement

Le fichier index.html a été créé pour servir de page d’accueil du site web, mais il est actuellement vide. L’étape suivante consiste à y ajouter du code HTML. Nous allons commencer par quelque chose de simple en utilisant l’éditeur intégré de Cloud Shell pour ajouter une seule ligne de code HTML.

  1. Ouvrez index.html dans l’éditeur en ligne en tapant code index.html à l’invite du terminal :

    code index.html

  2. Tapez ou collez les instructions suivantes dans l’éditeur :

    <h1>Our Feline Friends</h1>

  3. Enregistrez le fichier et fermez l’éditeur en ligne. Vous pouvez sélectionner les points de suspension « ... » dans le coin droit de l’éditeur Cloud Shell ou utiliser la touche de raccourci (Ctrl+S sur Windows et Linux, Cmd+S sur macOS).

  4. Utilisez la commande git status pour vérifier l’état de l’arbre de travail :

    git status

  5. Vous pouvez voir que Git est au courant des changements que vous avez apportés :

    On branch main
Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)
  (use "git checkout -- <file>..." to discard changes in working directory)

    modified:   index.html

no changes added to commit (use "git add" and/or "git commit -a")

  6. Maintenant, commitez les changements :

    git commit -a -m "Add a heading to index.html"

    Notez que nous n’avons pas exécuté la commande git add cette fois-ci pour indexer nos changements. Au lieu de cela, nous avons utilisé l’indicateur -a dans la commande git commit. L’option -a ajoute tous les fichiers que vous avez modifiés depuis le dernier commit. Elle n’ajoutera pas les nouveaux fichiers. Pour ajouter de nouveaux fichiers, vous avez toujours besoin de git add.

  7. Vérifiez la sortie. Elle doit être semblable à cet exemple :

    [main 8c9143a] Add a heading to index.html
 1 file changed, 1 insertion(+)

Le changement apporté à index.html a été commité. Il existe maintenant deux versions du fichier dans le dépôt, même si vous n’en voyez qu’une seule (la version actuelle). L’un des avantages de Git est qu’il vous permet de restaurer les changements que vous avez apportés, ou de remonter le temps pour voir les versions précédentes. Nous en saurons plus sur ce sujet important un peu plus tard.
